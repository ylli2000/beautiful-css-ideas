CSS Tips and Tricks

A collection of useful CSS techniques and best practices that can help you write more efficient and maintainable styles. These tips cover various aspects of CSS, from layout to animations and everything in between.

1. Text Wrapping with Balance

Browser Support: 87.5% (Chrome 114+, Edge 114+, Safari 16.5+, Firefox 116+)

The text-wrap: balance property helps create more aesthetically pleasing text blocks by balancing line lengths. This is particularly useful for headings and short paragraphs where you want to avoid single words on the last line.

.unbalanced-text {
    max-width: 50ch;
}

.balanced-text {
    max-width: 50ch;
    text-wrap: balance;
}

2. Font Size Interpolation with interpolate-size

Browser Support: 87.5% (Chrome 118+, Edge 118+, Safari 17+, Firefox 119+)

The interpolate-size property allows smooth font size transitions between size keywords like small, medium, and large. This is particularly useful for responsive design where you want text to scale fluidly with the viewport. 

:root {
    interpolate-size: allow-keywords;
}

.normal-text {
    font-size: medium; /* No interpolation between keywords */
}

.interpolated-text {
    font-size: calc(small + (large - small) * (100vw - 320px) / (1200 - 320));
}

/* Height transition example */
.height-transition-box {
    height: 2.5rem;
    overflow: hidden;
    transition: height 1s ease;
}

.height-transition-box:hover,
.height-transition-box:focus-within {
    height: max-content; /* Works with interpolate-size */
}

3. Input Field Sizing with field-sizing

Browser Support: 87.5% (Chrome 114+, Edge 114+, Safari 16.5+, Firefox 116+)

The field-sizing: content property allows input fields to automatically adjust their width based on their content. This is particularly useful for form inputs where you want the field to grow with the text.

.fixed-input {
    width: 200px;
}

.content-sized-input {
    field-sizing: content;
    min-width: 200px;
    max-width: 200px;
}

/* Textarea examples */
.fixed-textarea {
    width: 200px;
    min-height: 100px;
    max-height: 200px;
}

.content-sized-textarea {
    field-sizing: content;
    width: 200px;
    min-height: 100px;
    max-height: 200px;
}

4. CSS Anchor Positioning

Browser Support: 87.5% (Chrome 114+, Edge 114+, Safari 16.5+, Firefox 116+)

The CSS anchor() function allows elements to be positioned relative to other elements using named anchor points. This example demonstrates how to create a resizable infobox using anchor points and interactive drag controls. 

/* the % percentage means the distance from the anchor point
 to the edge of the infobox. It causes the gripping dot to be 
 landing half way in and half way out of the infobox border. */
.infobox {
    position-anchor: --myAnchorTL;
    position: fixed;
    top: anchor(--myAnchorTL 50%);
    left: anchor(--myAnchorTL 50%);
    bottom: anchor(--myAnchorBR 50%);
    right: anchor(--myAnchorBR 50%);
}

5. Logical Padding for Different Text Directions

Browser Support: 98.5% (Chrome 69+, Edge 79+, Safari 12.1+, Firefox 41+)

The padding-inline properties adapt to the text direction, making them ideal for multilingual layouts. This example shows how padding-inline-start and padding-inline-end work with different writing modes. 

.text-block {
    padding-inline-start: 2rem;
    padding-inline-end: 1rem;
    background: rgba(255, 255, 255, 0.1);
    border-radius: 4px;
}

/* RTL text */
.rtl-text {
    direction: rtl;
}

/* Vertical text */
.vertical-text {
    writing-mode: vertical-rl;
    text-orientation: upright;
}
